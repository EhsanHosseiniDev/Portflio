## Overview

Designed a Windows-based 3D visualization and control system for a custom parallel robot. The application imports SolidWorks CAD models and animates them in a WPF environment, reflecting real-time motion driven by user-defined parameters.

## Highlights

- **Inverse kinematics** layer translated end-effector targets into joint angles for the robot's parallel mechanism.
- **Bidirectional serial communication** with the robot's hardware board for live feedback during operation.
- **MVVM** for clean separation between the visualization, kinematics core, and UI.
- Architected so new robot configurations and hardware boards can be plugged in without rewriting the visualization layer.

## What I learned

Bridging real hardware with a polished UI required treating the visualization as a thin client over a robust kinematics + comms core. The discipline of designing for testability paid off heavily when the hardware misbehaved — most failure modes were reproducible without the robot in the loop.

## Reborn for the web

A decade later I'm rewriting this project's software layer from scratch — this time as a browser-based simulator, with a C#/WebAssembly computation core and a React Three Fiber 3D layer, no hardware required.
